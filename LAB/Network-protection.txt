Configure the iptables rule to allow SSH to your firewall only from LAN network
Log all ssh connections from LAN network with a tag "SSH from LAN"

Configure iptables rule to protect the public interface of the firewall from ping flood attack using limit module.
Example rules are 
sudo iptables -A INPUT -p icmp --icmp-type echo-request -m limit --limit 5/minute --limit-burst 3 -j ACCEPT
sudo iptables -A INPUT -p icmp --icmp-type echo-request -j DROP

Configure iptables rule to protect the public interface of the firewall from portscan. The example rules are as below.
iptables -A INPUT -p tcp --dport 1-65535 -m recent --set --name portscan
iptables -A INPUT -p tcp --dport 1-65535 -m recent --update --seconds 60 --hitcount 10 --name portscan -j DROP

Install nmap on the attacker machine and scan the Firewall public IP
yum install nmap -y
nmap -sS public-ip 

Configure the iptables rule to protect the forwarded port 80 against the SYN flood attack
Check if the website opens from the Windows browser.
Example rules are 
iptables -A FORWARD -p tcp --syn --dport 80 -m limit --limit 10/s --limit-burst 20 -j ACCEPT
iptables -A FORWARD -p tcp --syn --dport 80 -m limit --limit 1/min --limit-burst 5 -j LOG --log-prefix "SYN Flood Drop: "
iptables -A FORWARD -p tcp --syn --dport 80 -j DROP

From attacker machine use hping 3 to perform SYN flood

hping3 -S public-ip -p 80 --fast
also try 
hping3 -S public-ip -p 80 --flood

Try to open the website from your Windows machine while the hping3 attack is running.

Change the default policy of INPUT and FORWARD chain to DROP.
